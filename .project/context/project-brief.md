# Project Brief

## Problem
- Happy exists to let users monitor and take over Claude Code and Codex sessions away from their main machine without giving up encrypted transport or cross-device continuity.
- Delano is being used here to add local project contracts, delivery context, and repeatable validation around repository changes and bugfix work.

## Target Outcome
- Near term: make the Delano-backed `happy-bugfix-stabilization` effort usable for real execution instead of leaving it as starter scaffolding.
- Success means future agents can resume work from `.project/` context quickly, and Delano validation/runtime issues are visible instead of hidden.

## Scope Boundaries
- In scope: repository context, Delano operating context, package boundaries, runtime constraints, current health, and known Delano validation issues.
- Out of scope: inventing missing spec/plan details for `happy-bugfix-stabilization`, changing app/server behavior, or documenting personal/local machine setup.
