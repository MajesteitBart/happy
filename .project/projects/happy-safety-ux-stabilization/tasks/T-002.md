# Task: Add lifecycle diagnostics and recovery events

## Description

Implement content-free lifecycle events that let the app and CLI distinguish normal handoff, disconnect, abort, recovery, crash, and archived states. These events should power better recovery UX and future diagnostic bundles.

## Acceptance Criteria

- [ ] Introduce a structured lifecycle event shape with no message content or secrets.
- [ ] Emit events from local/remote switch, abort, process exit, disconnect, reconnect, and recovery paths.
- [ ] Persist or transmit the minimum metadata required to diagnose state divergence.
- [ ] Add tests for event classification where existing test harnesses allow it.

## Traceability

- Story: US-001
- Acceptance criteria: AC-001, AC-002

## Technical Notes

- Do not log raw prompts, tool outputs, auth tokens, or local absolute paths.
- Prefer a shared wire type if events cross app/server/CLI boundaries.

## Definition of Done

- [ ] Lifecycle events are emitted consistently.
- [ ] Tests or fixture traces cover normal and failure transitions.
- [ ] User-facing state can be derived from event type without string parsing.
