# Task: Implement server-issued auth nonces

## Description

Replace replayable client-generated auth challenges with a server-issued nonce flow. Nonces should be purpose-bound, short-lived, single-use, and included in signed material with server origin and client/device identity where available.

## Acceptance Criteria

- [ ] Add nonce request endpoint and storage with hash, expiry, and purpose.
- [ ] Update auth verification to consume nonce exactly once.
- [ ] Include nonce, server origin, auth purpose, and client/device identifier in signed material where feasible.
- [ ] Add replay, expiry, wrong-purpose, and wrong-origin tests.

## Technical Notes

- Preserve migration behavior for existing clients if needed, but do not leave replayable auth as the long-term default.

- [ ] Auth flow tests cover success and failure cases.
