# Task: Map session lifecycle transitions

## Description

Inventory the current local, remote, daemon, app, server, Claude, and Codex session transitions and turn them into an explicit lifecycle contract. The target lifecycle is `created -> local-active -> remote-active -> aborting -> recovering -> completed/error -> archived`, with documented transition inputs, process ownership, persistence requirements, and user-visible states.

## Acceptance Criteria

- [ ] Identify current transition entry points across app, CLI, daemon, server, Claude, and Codex code.
- [ ] Define allowed states and transitions, including ownership of input, process liveness expectations, and persistence rules.
- [ ] Document abort, disconnect, reconnect, and local/remote switch semantics.
- [ ] Add a short implementation note or contract document in the repo so later code changes have one source of truth.
