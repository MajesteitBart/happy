# Task: Require daemon control authorization

## Description

Generate a daemon control token, store it with strict local permissions, and require `Authorization: Bearer <token>` plus a non-browser custom header for local daemon control endpoints.

## Acceptance Criteria

- [ ] Generate a random token on daemon startup or state creation.
- [ ] Store token in daemon state with mode `0600` and parent directory `0700` where supported.
- [ ] Require authorization for list, spawn, stop session, session started, and stop daemon endpoints.
- [ ] Reject unauthenticated and browser-like requests in tests.
- [ ] Update CLI client calls to send the token.

## Traceability

- Story: US-005
- Acceptance criteria: AC-006

## Technical Notes

- Binding to `127.0.0.1` is useful but not sufficient as the only boundary.

## Definition of Done

- [ ] Unauthenticated daemon control requests fail.
- [ ] Authenticated CLI control still works.
- [ ] Token migration behavior is documented.
