# Task: Require daemon control authorization

## Description

Generate a daemon control token, store it with strict local permissions, and require `Authorization: Bearer <token>` plus a non-browser custom header for local daemon control endpoints.

## Acceptance Criteria

- [x] Generate a random token on daemon startup or state creation.
- [x] Store token in daemon state with mode `0600` and parent directory `0700` where supported.
- [x] Require authorization for list, spawn, stop session, session started, and stop daemon endpoints.
- [x] Reject unauthenticated and browser-like requests in tests.
- [x] Update CLI client calls to send the token.

## Traceability

- Story: US-005
- Acceptance criteria: AC-006

## Technical Notes

- Binding to `127.0.0.1` is useful but not sufficient as the only boundary.

## Definition of Done

- [x] Unauthenticated daemon control requests fail.
- [x] Authenticated CLI control still works.
- [x] Token migration behavior is documented.
